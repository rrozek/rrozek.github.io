    API Endpoints

    The following section defines the core HTTP and WebSocket API endpoints required to interface with the AFSDK, Credential Manager, & Signal Server. All endpoints are assumed to be authenticated via token/session key exchange unless otherwise noted.

    Base Path

    Base URL: https://api.wallet-signal.io/v1

    Identity & Wallet Endpoints

    POST /identity/import

    SDK Method Reference: import_seed(mnemonic)

    Imports a BIP39-compliant mnemonic and derives core wallet keys.

    Input:

    {
  "mnemonic": "string"
}

    Output:

    {
  "success": true,
  "pubkey": "string",
  "paycode": "string"
}

    POST /identity/recover

    SDK Method Reference: recover_identity(mnemonic, backups)

    Regenerates wallet identity and credentials using BIP39 mnemonic and optional timestamped credential backups.

    Input:

    {
  "mnemonic": "string",
  "backups": []
}

    Output:

    {
  "success": true,
  "recovered_contacts": 5
}

    GET /identity/status

    SDK Method Reference: get_my_pubkey(), get_my_payment_code()

    Returns current session status, public identity, and sfname.

    Output:

    {
  "pubkey": "string",
  "paycode": "string",
  "sfname": "string",
  "status": "active"
}

    POST /identity/register

    SDK Method Reference: register_name(sfname, paycode)

    • Implicit signature handling via get_my_pubkey() & sign_challenge()
    • Content-Type: application/json

    Backend Action: Upon receiving this request:

    1. The backend validates the signature using the provided pubkey.
    2. It checks that sfname is available and not registered.
    3. It records the following new identity entry in the backend:
    {
  "sfname": "alice.wallet",
  "pubkey": "0x...",
  "paycode": "PM8TJ...",
  "registered_at": "2025-08-18T10:00:00Z",
  "status": "pending"
}
    1. Then, it emits a write to the DeFi SignalIdentity contract to finalize the registration.
    2. Once confirmation is received from the DeFi contract (via an emitted IdentityRegistered event), the backend updates the record:

    Response:

    {
  "success": true,
  "sfname": "alice.wallet",
  "tx_hash": "0x..."
}

    DELETE /identity/reset

    SDK Method Reference: reset_identity()

    Clears local session, memory cache, and credential bindings.

    Output:

    {
  "success": true,
  "message": "Identity reset complete"
}

    Credential Management

    POST /credentials/sign

    SDK Method Reference: sign_master_credential() (should be added)

    Requests Authority to sign a MasterCredential.

    Input:

    {
  "credential_type": "master",
  "payload": {}
}

    Output:

    {
  "signed_credential": {},
  "signature": "string"
}

    POST /credentials/add-contact

    SDK Method Reference: add_contact(contact)

    Issues a ContactCredential after saving metadata. Adds a contact and issues a ContactCredential for encrypted communication.

    Input:

    {
  "contact_sfname": "string",
  "contact_pubkey": "string",
  "metadata": {}
}

    Output:

    {
  "credential_id": "string",
  "contact_added": true
}

    GET /credentials/contact/:sfname

    SDK Method Reference: get_contact_metadata(sfname) (should be added)

    Returns stored credential metadata for a contact.

    Output:

    {
  "sfname": "string",
  "pubkey": "string",
  "metadata": {},
  "last_seen": "2025-08-18T10:00:00Z"
}

    DELETE /credentials/revoke

    SDK Method Reference: revoke_contact(sfname) (should be added)

    Revokes an existing contact credential (soft delete).

    Input:

    {
  "sfname": "string",
  "reason": "string"
}

    Output:

    {
  "success": true,
  "revoked_at": "2025-08-18T10:00:00Z"
}

    Signal Routing - Off Chain Signaling Only

    POST /signal/send

    SDK Method Reference: send_signal(sfname, chain)

    Dispatches an encrypted message using StrategyContext evaluation.

    Input:

    {
  "recipient_sfname": "string",
  "message": "string",
  "chain": "ethereum"
}

    Output:

    {
  "signal_id": "string",
  "strategy_used": "nym",
  "estimated_delivery": "2025-08-18T10:05:00Z"
}

    GET /signal/inbox

    SDK Method Reference: get_signal_inbox() (Should be added)

    Pulls unread messages from the relay, inbox, or fallback cache.

    Output:

    {
  "messages": [
    {
      "id": "string",
      "sender": "string",
      "content": "string",
      "timestamp": "2025-08-18T10:00:00Z"
    }
  ]
}

    POST /signal/ack

    SDK Method Reference: ack_signal(signal_id) (Should be added)

    Acknowledges message delivery to remove from retry queue.

    Input:

    {
  "signal_id": "string"
}

    Output:

    {
  "success": true,
  "acknowledged_at": "2025-08-18T10:00:00Z"
}

    POST /signal/echo

    SDK Method Reference: test_signal_loopback(payload) (Optional utility method)

    Loopback diagnostic endpoint to test encrypted delivery round-trip.

    Input:

    {
  "payload": "string"
}

    Output:

    {
  "echo_response": "string",
  "round_trip_time": 150
}

    System Meta & Logging

    GET /meta/version

    SDK Method Reference: get_api_version() (Should be added)

    Returns current API version and build info.

    Output:

    {
  "version": "1.2.0",
  "build_date": "2025-08-18",
  "commit_hash": "abc123"
}

    GET /logs/recent

    SDK Method Reference: get_recent_logs() (Should be added)

    Returns recent signal send/receive or credential validation logs.

    Output:

    {
  "logs": [
    {
      "timestamp": "2025-08-18T10:00:00Z",
      "level": "info",
      "message": "Signal sent successfully",
      "signal_id": "string"
    }
  ]
}

    POST /logs/report

    SDK Method Reference: report_log(logEntry) (Should be added)

    Pushes error or telemetry logs from SDK clients for sync/debug.

    Input:

    {
  "log_entry": {
    "level": "error",
    "message": "string",
    "context": {}
  }
}

    Output:

    {
  "success": true,
  "log_id": "string"
}

    GET /meta/channels

    SDK Method Reference: get_available_routes() (Should be added)

    Lists available signaling strategies (e.g., nym, beacon, fallback) for client runtime decision.

    Output:

    {
  "channels": [
    {
      "name": "nym",
      "available": true,
      "latency_ms": 200
    },
    {
      "name": "beacon",
      "available": true,
      "latency_ms": 500
    },
    {
      "name": "fallback",
      "available": true,
      "latency_ms": 1000
    }
  ]
}
