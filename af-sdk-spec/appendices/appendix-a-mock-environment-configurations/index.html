    Appendix A: Mock Environment Configurations

    This mock environment reflects the runtime behavior of the full WalletSDK system within a non-production testnet or partner-integration sandbox. It enables developers to validate session bootstrapping, encrypted signal propagation, and credential issuance workflows in conditions that mirror production reliability while offering extended observability.

    Key Features

    • End-to-end Nym routing simulation
    • Mock token issuance via AuthService
    • Staging relay routing with metrics enabled
    • Credential Authority reachable over HTTP(S)
    • Wallet API connected to testnet nodes and ephemeral storage

    These mocks support end-user test flows like:

    1. Importing a mnemonic and generating a full identity structure
    2. Issuing and verifying ContactCredentials
    3. Sending and receiving encrypted signals
    4. Simulating delivery timeouts and retry logic
    5. Validating session lifecycle and WebSocket fallbacks

    Configuration Parameters

    This configuration simulates a live testing environment used during pre-production validation of WalletSDK components and Nym communication layers. Parameters are tuned to enable verbose logging, stateless API tests, and fault injection support.

    Service Endpoints

    • signalServer: WebSocket endpoint used for encrypted message transport

      {
  url: "wss://staging.api.wallet-signal.io/v1/signals",
  heartbeatInterval: 30000,
  reconnectAttempts: 5,
  enableMetrics: true
}
      * authService: Token-based identity authentication server

      {
  url: "https://staging.api.wallet-signal.io/v1/auth",
  tokenExpiry: 3600000, // 1 hour
  refreshThreshold: 0.8, // Refresh at 80% of expiry
  mockTokens: true
}
      * nymAuthority: Credential issuing/validation API

      {
  url: "https://staging.api.wallet-signal.io/v1/nym",
  mockCredentials: true,
  validationDelay: 100, // Simulate network latency
  enableFaultInjection: false
}
      * walletApi: Backend endpoint for transaction and balance queries

      {
  url: "https://staging.api.wallet-signal.io/v1/wallet",
  testnetMode: true,
  mockBalances: true,
  ephemeralStorage: true
}
      * logLevel: Adjusts verbosity for diagnostics

      {
  level: "debug",
  include: ["routing", "encryption", "auth"],
  exclude: [],
  remoteSync: true,
  retentionDays: 7
}

    Test Scenarios

    Identity Management Testing

    // Test identity import and key derivation
const testMnemonic = "abandon abandon abandon abandon abandon abandon abandon abandon abandon abandon abandon about";
const wallet = await sdk.importIdentity(testMnemonic);

// Verify key derivation
assert(wallet.publicKey.length === 66); // Compressed secp256k1
assert(wallet.paycode.startsWith("PM8T"));

    Signal Propagation Testing

    // Test encrypted signal sending
const signal = {
  recipient: "test.wallet",
  message: "Hello from mock environment",
  priority: "normal"
};

const result = await sdk.sendSignal(signal);
assert(result.status === "delivered");
assert(result.strategy === "mock_nym_route");

    Fault Injection Testing

    // Simulate network timeout
mockEnvironment.injectFault({
  type: "network_timeout",
  duration: 5000,
  probability: 0.3
});

// Test retry logic
const signal = await sdk.sendSignal(testSignal);
await mockEnvironment.waitForRetry();
assert(signal.retryCount > 0);

    Environment Setup

    Docker Configuration

    version: '3.8'
services:
  mock-wallet-sdk:
    image: matterfi/mock-wallet-sdk:latest
    ports:
      - "8080:8080"
    environment:
      - NODE_ENV=staging
      - LOG_LEVEL=debug
      - FAULT_INJECTION_ENABLED=true
    volumes:
      - ./config:/app/config

    Local Development Setup

    # Clone and setup
git clone https://github.com/matterfi/mock-wallet-sdk.git
cd mock-wallet-sdk
npm install

# Configure environment
cp .env.example .env
# Edit .env with staging endpoints

# Start mock services
npm run start:staging

    Validation Workflows

    Session Bootstrap Validation

    1. Import Identity: Verify mnemonic parsing and key derivation
    2. Token Acquisition: Test authentication flow with mock tokens
    3. WebSocket Connection: Validate handshake and heartbeat
    4. Credential Issuance: Test contact credential creation and verification

    Signal Delivery Validation

    1. Message Encryption: Verify AES-GCM encryption/decryption
    2. Routing Strategy: Test strategy selection and fallback logic
    3. Delivery Confirmation: Validate acknowledgment handling
    4. Retry Logic: Test timeout and reconnection scenarios

    Error Simulation

    1. Network Failures: Simulate connection drops and timeouts
    2. Authentication Errors: Test token expiry and refresh flows
    3. Decryption Failures: Verify error handling for invalid keys
    4. Rate Limiting: Test throttling and backoff behavior
